# TU+ Stack

## Purpose

This subfolder hosts the emerging LLM-paralleled awareness-support architecture within the broader pre-metric extensions branch.

The working claim is that a useful awareness-support stack may require three distinct layers:

1. **TU** — a mindless choreography mapper  
2. **TU+** — a specialized choreography-aware predictor / replay / comparison layer  
3. **LLM** — a symbolic neocortex-like interpreter

This folder exists to make that stack explicit, modular, and testable.

---

## Why this subfolder exists

The broader `pre_metric_extensions` branch explores pre-metric guidance, temporal units, and choreography-sensitive architectures in general.

This subfolder narrows that work to a more specific path:

- how a temporal unit might map unfolding choreographies
- how an intermediate TU+ layer might support awareness
- how a higher symbolic LLM might interpret that field
- how video might serve as the first practical 4D input path

This is therefore the branch-inside-the-branch for the TU / TU+ / cortex architecture.

## Central idea

The current architecture is:

- **TU**  
  maps traces, trains, couplings, decay, restart, and field structure

- **TU+**  
  compares current choreography trains with stored patterns, predicts likely continuation, and emits awareness-support signals

- **LLM**  
  interprets significance, reasons symbolically, plans, and critiques the TU/TU+ mapping

A compact formulation:

> TU maps choreography structure. TU+ recognizes, predicts, and compares choreography structure. The cortex-like LLM interprets its meaning.

## First practical route

The strongest first test path currently appears to be:

- video input
- TU choreography mapping
- TU+ predictive / replay comparison
- LLM interpretation
- comparison back to the source video
- refinement of the mapping stack

This makes video the current best candidate for first serious input because it already supplies:
- time
- simultaneity
- persistence
- interruption
- drift
- recurrence
