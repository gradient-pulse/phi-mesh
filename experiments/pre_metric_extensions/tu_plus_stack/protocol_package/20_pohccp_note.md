# T-Protocol — PoHCCP Note v1

## Purpose

This note defines the role of **PoHCCP** inside T-Protocol.

PoHCCP stands for:

> **Principle of Highest Contextual Coherence Path**

Its purpose is to clarify the deepest candidate rule underlying TU+ continuation sensing.

This note exists because TU+ should not be misunderstood as:
- an ordinary predictor
- a reflex forecaster
- a generic planner
- or a next-token extrapolator

The protocol requires a deeper account of how likely continuation is sensed.

PoHCCP is the current best formulation of that account.

---

## Core principle

TU+ does not merely predict by extrapolation.

It senses likely continuation by favoring the path that preserves the **highest contextual coherence** available in the present transforming field.

Compactly:

> TU+ intuits likely continuation by tracking the path of highest contextual coherence through transformation.

---

## Why PoHCCP is needed

Without a deeper rule, TU+ prediction can be mistaken for:

- statistical continuation
- model-predictive simulation
- generic reasoning ahead
- weighted guesswork over future options

But TU+ is not choosing a next step in a vacuum.

It is balancing:

- stored choreography memory
- dynamic memory already present in sustained live choreography
- current coherence state
- mismatch history
- bounded contextual bias

That balancing requires a governing principle.

PoHCCP names that principle.

---

## Definition

**PoHCCP** is the principle that, among available continuations, the most viable continuation is the one that preserves the highest contextual coherence through present transformation.

This does not mean:
- the smoothest path in a superficial sense
- the most familiar path
- the most convenient symbolic interpretation
- the least surprising outcome
- the shortest route to closure

It means:

- the continuation that best preserves viable relation
- under the actual conditions of the current field
- while remaining answerable to mismatch and returned evidence

Compactly:

> PoHCCP selects not the easiest path, but the most coherence-preserving path available in context.

---

## Contextual coherence

Contextual coherence is not a single scalar convenience score.

In the protocol, it refers to the viability of the current field as a structured whole, including:

- train persistence
- coupling viability
- branch integrity
- contradiction sensitivity
- mismatch accountability
- restart intelligibility
- bounded symbolic guidance without overwrite

So when TU+ senses likely continuation, it is not simply asking:

- what comes next most often?

It is asking:

- which continuation best preserves the structured viability of the present field?

---

## Why “contextual” matters

PoHCCP is not a universal fixed-path rule detached from circumstances.

It is contextual because the coherence-preserving path depends on:

- the current field structure
- the current mismatch pressure
- the current persistence and decay conditions
- the current coupling or fragmentation state
- the present phase of unfolding
- the bounded influence of symbolic context where relevant

The best continuation is therefore not fixed in advance.

It is field-dependent.

Compactly:

> The highest-coherence path is always the highest-coherence path in this field, under these conditions, at this moment of transformation.

---

## Why “through transformation” matters

PoHCCP is not a static equilibrium rule.

It is a rule for unfolding systems.

The protocol does not operate over frozen states alone.  
It operates over changing trains, couplings, mismatches, and returns.

So the relevant question is not:

- which state looks best in isolation?

but:

- which transforming continuation preserves coherence best as the field changes?

This is why TU+ is continuity-sensitive rather than merely state-sensitive.

## PoHCCP and TU+

TU+ is the protocol layer most directly associated with PoHCCP.

TU+:
- compares current trains against stored choreography memory
- reads dynamic memory present in sustained live choreography
- detects mismatch and instability
- preserves unresolved alternatives where warranted
- projects candidate next trains

The deepest candidate rule beneath those activities is:

> favor the continuation that preserves the highest contextual coherence available in the transforming field.

This is why TU+ is better described as a continuation-sensing layer than as an ordinary predictor.

---

## PoHCCP and dynamic memory

PoHCCP depends on the fact that sustained choreography is itself dynamic memory.

A field does not confront the future as an empty present.

It confronts it through:
- persistent trains
- coupling history
- unresolved branches
- mismatch history
- bounded contextual pressure
- dynamic continuity already carried by the unfolding itself

So TU+ does not infer continuation from scratch.

It senses it by balancing:
- stored memory
- live dynamic memory
- present coherence pressure

PoHCCP is the rule by which that balancing becomes directional.

Compactly:

> Dynamic memory gives the field continuity; PoHCCP gives that continuity directional preference.

---

## PoHCCP and ambiguity

PoHCCP does not imply premature closure.

When multiple candidate continuations remain viable, the protocol may preserve ambiguity rather than forcing one winner.

Why?

Because under some conditions, the highest contextual coherence path is not yet singularly identifiable.

In such cases, coherence is preserved better by:
- keeping branches explicit
- preserving contradiction
- delaying symbolic commitment

So PoHCCP supports both:
- weighted preference
and
- lawful ambiguity preservation

That is one reason it differs from simplistic prediction.

---

## PoHCCP and mismatch

PoHCCP is not a guarantee of correctness in advance.

It is a principle of disciplined continuation sensing under correction pressure.

Returned traces may later show that the previously favored path:
- weakened
- bifurcated
- collapsed
- reopened
- or required restart

This does not invalidate PoHCCP.

It shows that contextual coherence is an unfolding matter, not a static certainty.

PoHCCP therefore remains answerable to:
- mismatch
- de-confirmation
- revision
- restart

Compactly:

> PoHCCP guides continuation sensing, but returned evidence still decides whether the path remains viable.

> PoLA may be the metric trace; PoHCCP names the deeper coherence-selective rule in protocol terms.
