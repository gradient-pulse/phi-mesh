# T-Protocol — Disclosure Map

## Purpose

This file defines the current disclosure discipline for T-Protocol materials.

Its purpose is to separate:

- what may be shared publicly
- what should only be shared under protected review
- what should only be shared under license
- what should remain internal

This is a working disclosure map and should be updated as the package matures.

---

## 1. Public conversation layer

These materials may be used in first-contact and ordinary strategic conversations.

Typical uses:
- investor conversations
- industry conversations
- introductory meetings
- high-level category explanation

### Included materials
- short conversation note
- introductory architecture brief
- one-page architecture note
- public-safe diagram
- high-level use-case framing

### Disclosure rule
These materials should explain the object clearly without being sufficient to reconstruct a faithful implementation.

---

## 2. Protected protocol review layer

These materials should only be shown under confidentiality, NDA, or another explicit protected-review condition.

Typical uses:
- serious technical evaluation
- partner diligence
- investor review after real interest
- industrial feasibility discussions

### Included materials
- detailed front-package sections
- role specifications
- bounded-flow rules
- feedback discipline
- operating modes in practical detail
- compliance logic
- selected support materials from the back package

### Disclosure rule
Only disclose what is necessary for evaluation.  
Do not disclose full implementation-grade material casually.

---

## 3. Licensed implementation layer

These materials should only be provided under a valid license or equivalent formal agreement.

Typical uses:
- actual implementation
- pilot deployment
- commercial use
- bounded faithful-use rights

### Included materials
- full protocol package
- approved implementation materials
- deployment-specific guidance
- compliance procedures
- exact or near-operational instruction materials required for faithful use

### Disclosure rule
These materials constitute the implementation-grade substance of T-Protocol and should not be distributed outside licensed conditions.

---

## 4. Internal-only layer

These materials remain internal unless selectively surfaced for protected review.

Typical uses:
- origin preservation
- legal preparation
- trust and diligence support
- internal continuity
- evidence preservation

### Included materials
- broader observation archive
- drafting history
- internal debate notes
- deeper RGPx linkage materials not required for ordinary use
- package-planning notes
- legal preparation notes still under development

### Disclosure rule
These materials are not part of ordinary outward-facing circulation.

---

## 5. Practical test

A practical rule for classification is:

> If a document would allow a capable team to reconstruct a meaningful version of T-Protocol, it should not be treated as ordinary public material.

---

## 6. Current package intent

### Public
- explain the category
- establish seriousness
- open conversations

### Protected
- evaluate the category
- test technical coherence
- support diligence

### Licensed
- enable faithful use
- preserve protocol identity
- preserve commercial value

### Internal
- preserve origin
- preserve evidence
- preserve continuity

---

## 7. Status

This is a working map.

It should be updated as:
- more front-package files are stabilized
- protected-review materials are clearly separated
- licensed implementation materials are defined
- legal and IP review progresses
