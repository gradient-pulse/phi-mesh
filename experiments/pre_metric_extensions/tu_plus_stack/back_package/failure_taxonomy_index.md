# T-Protocol — Failure Taxonomy Index

## Purpose

This index tracks the failure-taxonomy materials that name and organize the principal ways T-Protocol may fail, drift, be misread, or be weakly packaged.

The taxonomy exists to sharpen:
- testing
- diagnosis
- specification
- compliance logic
- disclosure discipline

---

## What the failure taxonomy supports

Failure materials support claims about:
- role-boundary integrity
- regime-reading discipline
- perturbation boundedness
- recovery integrity
- second-order robustness
- endurance interpretation
- packaging discipline

---

## Current failure groupings

### A. Role-boundary failures
These include:
- TU semantic contamination
- TU+ inflation
- cortexLLM overreach
- premature role fusion
- boundary drift without collapse

### B. Regime-classification failures
These include:
- false unity
- false fragmentation
- false restart
- false degradation
- false containment
- false recovery
- false second-order reassurance
- false residue-contact fusion

### C. Perturbation-response failures
These include:
- immediate collapse
- unbounded weakening
- spread beyond local region
- containment failure
- supervisory distortion failure
- continuity robustness failure
- longitudinal robustness failure
- second-order bounded-response failure

### D. Measurement and interpretation failures
These include:
- overfitting to one perturbation class
- repetition mistaken for discovery
- weak evidence fossilized as law too early
- packaging ahead of specification

### E. Endurance-interpretation failures
These include:
- saturation mistaken for boundary contact
- saturation mistaken for unlimited robustness
- mechanism misframing
- society-of-agents substitution
- inferential-drift horizon neglect

---

## Why this matters

The failure taxonomy is important because a licensable protocol must eventually specify not only:
- what counts as valid behavior

but also:
- what counts as invalid behavior
- what counts as degraded interpretation
- what counts as misapplication or misuse
